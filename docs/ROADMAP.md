## Phase 1: 프로젝트 초기 설정

**예상 소요 시간**: 1-2일  
**목표**: 견고한 개발 기반 구축 및 Notion 연동 환경 완성

### 주요 작업 항목

- [ ] **Next.js 프로젝트 구조 설정**
  - Next.js 15 + TypeScript + Tailwind CSS 초기화
  - 프로젝트 폴더 구조 생성 (app, components, lib, types 등)
  - 기본 레이아웃 구조 구성

- [ ] **Notion API 연동 환경 구축**
  - `@notionhq/client` 패키지 설치
  - 환경 변수 설정 (.env.local)
  - Notion 데이터베이스 생성 및 API 키 발급
  - 기본 테스트 글 3개 이상 작성

- [ ] **타입 정의 및 유틸리티 기본 구조**
  - `types/index.ts` 생성 (Post, Category, PageBlock 등)
  - `lib` 폴더 구조 설정

- [ ] **기본 레이아웃 컴포넌트**
  - Header 컴포넌트 (로고, 네비게이션)
  - Footer 컴포넌트
  - 메인 레이아웃 래퍼

### 완료 기준

- ✅ npm 설치 및 개발 서버 정상 실행 (`npm run dev`)
- ✅ `npm run type-check` 실행 시 에러 없음
- ✅ Notion 데이터베이스 생성 완료
- ✅ Notion API 키 발급 및 `.env.local`에 설정
- ✅ 기본 Header/Footer 컴포넌트 렌더링 확인
- ✅ `localhost:3000` 접속 시 에러 없이 페이지 로드

### 이유

견고한 기반 없이는 기능 개발이 어렵고, 추후 리팩토링 비용이 증가합니다.  
초기에 타입 정의와 폴더 구조를 잘 정의하면 팀/개인 프로젝트 확장이 수월합니다.

---

## Phase 2: 공통 모듈 개발

**예상 소요 시간**: 2-3일  
**목표**: 모든 기능에서 재사용되는 공통 코드 구현

### 주요 작업 항목

- [ ] **Notion API 공통 함수 구현** (`lib/notion-client.ts`)
  - Notion 클라이언트 인스턴스 생성
  - `fetchPages()` - 글 목록 조회 (필터링, 정렬)
  - `fetchPageContent()` - 페이지 상세 콘텐츠 조회
  - `getCategories()` - 카테고리 목록 조회
  - 에러 핸들링 및 타입 안정성 확보

- [ ] **Notion 블록 렌더러 구현** (`lib/notion-block-renderer.ts`)
  - 제목 (heading1, heading2, heading3)
  - 단락 (paragraph)
  - 리스트 (bulleted_list_item, numbered_list_item)
  - 코드 블록 (with 문법 강조)
  - 이미지 및 인용
  - 테이블 및 구분선

- [ ] **공통 UI 컴포넌트 개발** (`components/ui/`)
  - Card 컴포넌트 (글 목록용)
  - Button, Badge, Tag 컴포넌트
  - Pagination 컴포넌트
  - Loading, Error 상태 컴포넌트

- [ ] **공통 컴포넌트** (`components/common/`)
  - 검색 입력 컴포넌트
  - 카테고리 필터 컴포넌트
  - 메타데이터 표시 컴포넌트

- [ ] **유틸리티 함수 작성** (`lib/utils.ts`)
  - 날짜 포맷팅 함수
  - 텍스트 잘라내기 (truncate)
  - CSS 클래스 병합 함수
  - URL 생성 함수

### 완료 기준

- ✅ Notion API 함수 모두 구현 및 테스트
- ✅ `npm run type-check` 에러 0개
- ✅ 모든 Notion 블록 타입 렌더링 가능 (테스트 글로 확인)
- ✅ UI 컴포넌트 스토리북 또는 임시 테스트 페이지에서 확인
- ✅ 유틸리티 함수 모두 구현

### 이유

모든 기능에서 재사용되는 코드를 먼저 만들어야 중복 방지합니다.  
특히 Notion 콘텐츠 렌더러는 상세 페이지, 미리보기 등 여러 곳에서 필요합니다.

## Phase 3: 핵심 기능 개발

**예상 소요 시간**: 3-4일  
**목표**: 블로그의 가장 기본이 되는 기능 완성

### 주요 작업 항목

- [ ] **블로그 글 목록 페이지** (`app/page.tsx`)
  - 글 목록 조회 및 렌더링
  - 글 카드 표시 (제목, 카테고리, 발행일, 요약, 썸네일)
  - 최신순 정렬 (발행일 기준)
  - 페이지네이션 구현 (페이지당 10개 글)
  - "발행됨" 상태의 글만 표시

- [ ] **블로그 글 상세 페이지** (`app/posts/[id]/page.tsx`)
  - 글 메타데이터 표시 (제목, 작성일, 카테고리, 태그)
  - Notion 콘텐츠 풀 렌더링
  - 썸네일 이미지 표시
  - 뒤로가기 네비게이션
  - 동적 라우팅 처리

- [ ] **API 라우트 구현**
  - `GET /api/posts` - 글 목록 조회 (페이지네이션, 필터링 지원)
  - `GET /api/posts/[id]` - 글 상세 조회

- [ ] **SSG/ISR 설정**
  - 글 목록 페이지 ISR 설정 (revalidate: 3600)
  - 글 상세 페이지 SSG 또는 ISR 설정

- [ ] **반응형 디자인** (모바일, 태블릿, 데스크톱)
  - 모든 페이지 모바일 최적화
  - 터치 친화적 UI

### 완료 기준

- ✅ 홈 페이지 (`/`) 로드 시 글 목록 표시
- ✅ 글 카드 클릭 시 상세 페이지로 이동
- ✅ 상세 페이지에서 Notion 콘텐츠 정상 렌더링
- ✅ 모든 페이지 로딩 시간 < 3초 (Lighthouse 기준)
- ✅ 모바일/태블릿/데스크톱 모두 정상 표시
- ✅ 페이지네이션 정상 작동

### 이유

글 목록과 상세 페이지는 블로그의 가장 기본이 되는 기능입니다.  
이 두 기능이 완성되어야 검색, 필터링 등 추가 기능을 구현할 수 있습니다.

## Phase 4: 추가 기능 개발

**예상 소요 시간**: 2-3일  
**목표**: 사용자 경험을 향상시키는 부가 기능 구현

### 주요 작업 항목

- [ ] **카테고리별 필터링** (`app/category/[name]/page.tsx`)
  - 카테고리 목록 조회 및 표시
  - 선택된 카테고리의 글만 필터링
  - 카테고리 페이지 생성
  - 상세 페이지에서 카테고리 클릭 시 필터링 페이지로 이동

- [ ] **검색 기능** (`app/search/page.tsx`)
  - 글 제목 및 태그 기반 검색
  - 실시간 검색 결과 표시
  - 검색 결과 하이라이팅
  - 검색 입력 박스 (고정 위치)

- [ ] **SEO 최적화**
  - 메타 태그 설정 (title, description, og:image)
  - Sitemap 생성
  - robots.txt 설정
  - 구조화된 데이터 (schema.org)

- [ ] **성능 최적화**
  - 이미지 최적화 (Next.js Image 컴포넌트)
  - 폰트 최적화
  - 코드 분할 (dynamic imports)
  - 캐싱 전략 개선

- [ ] **UI/UX 개선**
  - 다크/라이트 테마 전환 (선택사항)
  - 로딩 스켈레톤 UI
  - 에러 상태 페이지 (404, 500)

### 완료 기준

- ✅ 카테고리별 필터링 정상 작동
- ✅ 검색 기능으로 글 찾기 가능
- ✅ Lighthouse SEO 점수 > 90점
- ✅ Lighthouse Performance 점수 > 80점
- ✅ 에러 페이지 (404, 500) 구현

### 이유

핵심 기능이 완성된 후 사용자 경험과 검색 엔진 최적화를 추가합니다.  
SEO 최적화는 블로그의 가시성에 매우 중요합니다.
